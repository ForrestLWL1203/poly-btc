# Dashboard frontend

React UMD + precompiled `app.js`. Edit `app.jsx`, then run `dashboard/web/build.sh` before deployment.

```
dashboard/web/
├── index.html      entry (loads React/ReactDOM vendor + app.js)
├── app.css         glassmorphism design system (tokens from hyper/docs/design_handoff README §7)
├── app.jsx         shell: login, navigation, observer controls, page routing
├── components/     dashboard pages and page-specific subcomponents
├── lib/            API client, formatting helpers, icons, refresh/SSE layer
├── app.js          precompiled bundle generated by dashboard/web/build.sh
├── vendor/         React/ReactDOM UMD bundles committed for VPS deploy
└── dev/            preview-only helpers (NOT used in production)
    ├── seed_mock.py     seed a realistic mock data/hl_mock.db
    └── mock_consumer.py runs the REAL observer command loop + a scanner/rolling simulator
```

## Run the preview

1. **Vendor libs**. React/ReactDOM are committed because `index.html` needs them on the VPS:
   ```
   mkdir -p dashboard/web/vendor && cp /path/to/poly-fight/poly_fight/dashboardV2/vendor/{react-18.3.1.production.min.js,react-dom-18.3.1.production.min.js} dashboard/web/vendor/
   ```
2. **Seed mock data:** `python3 dashboard/web/dev/seed_mock.py data/hl_mock.db`
3. **Run the mock consumer** (so UI actions + rolling/rescan status are live):
   `python3 dashboard/web/dev/mock_consumer.py data/hl_mock.db`
4. **Serve** (static + API): `DASH_PASSWORD=mock123 python3 -m dashboard.server --db data/hl_mock.db --static dashboard/web --port 8810`
   (or use `.claude/launch.json` → name `dashboard`). Open http://127.0.0.1:8810 (auto-logs in with `mock123`).

## Against the REAL system
Point `--db` at the live `data/hl.db` (fed by the real observer/scanner) and drop the mock consumer.
The frontend already speaks the real API contract and uses SSE with polling fallback for live refresh.
